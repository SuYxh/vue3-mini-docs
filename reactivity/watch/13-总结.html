13-总结 ​

那么到这里，咱们整个的 响应系统 就全部讲解完成了。整个响应系统我们分成了：

  1. reactive
  2. ref
  3. computed
  4. watch

四大块来进行分别的实现。

通过之前的学习可以知道，响应式的核心 API 为 Proxy。整个 reactive 都是基于此来进行实现。

但是 Porxy 只能代理 复杂数据类型，所以延伸除了 get value 和 set value 这样 以属性形式调用的方法， ref 和 computed 之所以需要 .value 就是因为这样的方法。
